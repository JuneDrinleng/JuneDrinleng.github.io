## 1 First introduction to arrays

For some application scenarios, for example, we need to calculate the average and also need to give the numbers that are greater than the average. At this time, the numbers we enter need to be recorded. At this time, setting many variables is often inappropriate (a stupid way). How to deal with this situation is the creation of arrays.

~~~java
int[] numbers =new int[100]
~~~

Create an array named `numbers` that can hold 100 `int`

Specifically, fill it in with numbers:

~~~
numbers[index]=values_index
~~~

## 2 Create array

An array is a container whose size cannot be changed once created; all elements in it are of the same type.

For example, for the `numbers` array in 1:

~~~java
int[] numbers =new int[100]
~~~

After creation, only int can be stored, and only 100 can be stored.

The format created is as follows

~~~java
<type>[]<array name>=new <type of elements in array>[number of elements in array]
~~~

It should be noted that the index of the array starts from 0

## 3 Elements of array

Accessing the elements of the array is implemented using indexes. For example, we want to call out the fifth element of the array `numbers`:

~~~java
numbers[4]
~~~

You can also assign a value to it:

~~~java
numbers[4]=11
~~~

It should be noted that even if your index exceeds the index boundary of the array, the compiler will not check whether it is valid when writing the code. You will only know that your index exceeds the index boundary when running.

Therefore, it is dangerous to set the size of the array when the number of numbers entered by the user is unknown. We should set the size of the array according to the number of numbers read in by the user, thus avoiding the problem of overflowing the index.

Arrays have an inherent property, which is length:

~~~java
for (int i=0;i<grade.length;i++){
	operation
}
~~~

`grade` here is an array

## 4 Array variables

In addition to initializing first and then assigning values one by one (new method), you can also assign values directly during initialization:

~~~java
int[] scores={97,87,97,88,67,22}
~~~

It should be noted that this is because ordinary variables correspond to the value of the variable, but for arrays, the array and data are just pointers:

So when `int[]b=a`, it is equivalent to b also establishing an index pointing to the same position as a:

Then changing the 0th position of the data through the b index will also change under the a index.

## 5 task: Voting statistics

~~~java
import java.util.Scanner;

public class VoteStatistic {
    public static void main(String[] args) {
        Scanner in = new Scanner(System.in);
        int x;
        int[] numbers =new int[10];
        x=in.nextInt();
        while(x!=-1){
            if(x>=0 && x<=9){
                numbers[x]++;
            }
            x=in.nextInt();
        }
        for(int i=0;i< numbers.length;i++){
            System.out.println(i+" : "+numbers[i]);
        }
    }
}

~~~

Directly create an array with the number of occurrences of input [0,9]

## 6 Traverse the array

![image-20250321205734407](https://raw.githubusercontent.com/JuneDrinleng/JuneDrinleng.github.io/main/assets/img/image-20250321205734407.png)

Generally, a for loop starting from 0 and traversing to length is used to traverse.

In addition to this you can:

~~~java
for (int k:data){
	operation
}
~~~

This is the for-each loop. The k in each loop is the content in the data. It is particularly suitable for traversing every content in the array, but the disadvantage is that it does not know the index position of k. It should be noted that if this is:

~~~java
for (int k:data){
	k=9;
}
~~~

The contents of the array cannot be changed here

In addition, there are other methods:

~~~java
boolean[] isprime = new boolean[100];
for (int i=0;i<isprime.length;i++){
	isprime[i]=true;
}
for (int i=2; i<isprime.length;i++){
	if (isprime[i]){
		for (int k=2;i*k<isprime.length;k++){
			isprime[i*k]=false;
		}
	}
}
for (int i=2;i<isprime.length;i++){
    if (isprime[i]){
        System.out.println(i);
    }
}
~~~

That is to say, all multiples of prime numbers are not prime numbers, and the rest are prime numbers.

import java.util.Scanner;

public class board {
    public static void main(String[] args) {
        Scanner in = new Scanner(System.in);
        final int Size=3;
        int [][] board= new int[Size][Size];
        for (int i=0;i<Size;i++){
            for (int j=0;j<Size;j++){
                System.out.print("Enter the value of board line"+(i+1)+" and column"+(j+1)+":");
                board[i][j]=in.nextInt();
            }
        }
        MainLOOP:
        for (int i=0;i<Size;i++){
            int const_num=board[i][0];
            for (int j=0;j<Size;j++){
                if (board[i][j]!=const_num){
                    break;
                }
                else if (board[i][j]==const_num && j==Size-1){
                    System.out.print(const_num+" wins ");
                    break MainLOOP;
                }
            }
        }
        MainLOOP2:
        for (int j=0;j<Size;j++){
            int const_num=board[0][j];
            for (int i=0;i<Size;i++){
                if (board[i][j]!=const_num){
                    break;
                }
                else if (board[i][j]==const_num && i==Size-1){
                    System.out.print(const_num+" wins ");
                    break MainLOOP2;
                }
            }
        }
        int const_num=board[0][0];
        for (int i=0;i<Size;i++){
            if (board[i][i]!=const_num){
                break;
            }
            else if (board[i][i]==const_num && i==Size-1){
                System.out.print(const_num+" wins ");

            }

            }

            }
        }
    }
