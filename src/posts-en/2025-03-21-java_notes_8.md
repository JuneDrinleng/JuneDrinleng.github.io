## 1 Definition of function

Some code contents will be used repeatedly. In order to avoid repetition every time, we choose the form of constructor function, so that we only need to call this function every time.

For example, in judging prime numbers:

![image-20250321230346931](https://raw.githubusercontent.com/JuneDrinleng/JuneDrinleng.github.io/main/assets/img/image-20250321230346931.png)

You can set the name of a function to `isPrime`

Calculate three sets of sums, namely 1~10, 20~30, and 35~45. How to implement it?

To avoid code duplication, we can construct the function

~~~
public static void sum(int a, int b){
	int i;
	int sum=0;
	for (i=0;i<=b;i++){
		sum=sum+i;
	}
	System.out.println(the sum of a+" to "+b+" is "+sum);
}
~~~

Then you only need `sum(1,10)`, `sum(20,30)`, `sum(35,45)`

**So what is a function? **

A function is a piece of code that can receive several parameters, then do one thing, and may finally return a value of 0 or 1.

When defining a function, `public static void sum(int a, int b)` is the **function header**, where `sum` is the **function name**, `void` is the **return type**, what is in the curly brackets is the **function body**, and the content of the parentheses after the function name is the **parameter list** of the function

## 3 Function call

Calling a function is implemented using `function name (parameter value)`. Even if the function has no parameters, these parentheses are necessary.

When there are parameters, the correct order of the parameters must also be given, that is, the parameters are passed according to the order of the defined parameters.

A function always knows where it was called from and where to return to

To return a value in a function, use `return`

If you want `return`, then when defining the function, you need to write the return type before the function name:

![image-20250321233136884](https://raw.githubusercontent.com/JuneDrinleng/JuneDrinleng.github.io/main/assets/img/image-20250321233136884.png)

Here the parameters can be passed directly `max(c,23)`. It is a bad design for a function to have multiple exits.

The `void` function does not return a value. If you want to return, you can only use `return;`. It is used under certain conditions to jump out of the function when it reaches this point.

## 4 Local variables of functions

Each run of the function creates an independent space, which has nothing to do with the internal variables of the last run.

Variables defined in functions are local variables

![image-20250321233824415](https://raw.githubusercontent.com/JuneDrinleng/JuneDrinleng.github.io/main/assets/img/image-20250321233824415.png)

Within a scope, once a variable is defined, a variable with the same name cannot be defined again.

### **Rules for local variables**

- Local variables are defined within a block.

  - It can be defined inside a **function** block.
- Can also be defined within a **statement** block.
  - You can even use a pair of curly braces to define variables.

- Before the program enters this block, the variables in it **do not exist**. After leaving this block, the variables **disappear**.
