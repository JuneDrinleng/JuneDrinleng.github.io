## 1 Function Definition

Some code content is reused repeatedly. To avoid repetition, functions are constructed. This way, we only need to call the function each time

For example, in prime number judgment:

![image-20250321230346931](https://raw.githubusercontent.com/JuneDrinleng/JuneDrinleng.github.io/main/assets/img/image-20250321230346931.png)

We can define a function named `isPrime`

To calculate three sums: 1~10, 20~30, and 35~45, how to implement this?

To avoid code duplication, we can construct a function

~~~
public static void sum(int a, int b){
	int i;
	int sum=0;
	for (i=0;i<=b;i++){
		sum=sum+i;
	}
	System.out.println(a+" to "+b+" sum is "+sum);
}
~~~

Then simply call `sum(1,10)`, `sum(20,30)`, and `sum(35,45)`

**What is a function?**

A function is a block of code that can receive several parameters, perform a task, and may return 0 or 1 value

When defining a function, `public static void sum(int a, int b)` is the **function header**, where `sum` is the **function name**, `void` is the **return type**, and the content within the curly braces is the **function body**. The content inside the parentheses after the function name is the **parameter list**

## 3 Function Calls

Function calls are implemented using the format `functionName(parameterValues)`. Even if a function has no parameters, the parentheses are still required

When parameters are present, the correct order of parameters must be provided according to the defined parameter order

A function always knows where it was called from and where to return to

To return a value in a function, use `return`

If `return` is used, the return type must be specified in the function definition before the function name:

![image-20250321233136884](https://raw.githubusercontent.com/JuneDrinleng/JuneDrinleng.github.io/main/assets/img/image-20250321233136884.png)

Here, parameters can directly be passed as `max(c,23)`. Having multiple exits in a function is poor design

`void` functions do not return values. If `return` is used, it can only be `return;` to exit the function under certain conditions

## 4 Local Variables in Functions

Each function run creates an independent space, unrelated to the internal variables from previous runs

Variables defined inside a function are local variables

![image-20250321233824415](https://raw.githubusercontent.com/JuneDrinleng/JuneDrinleng.github.io/main/assets/img/image-20250321233824415.png)

Within a scope, once a variable is defined, a variable with the same name cannot be redefined

### **Rules for Local Variables**
- Local variables are defined within a block.

  - It can be defined within a **function** block.
- They can also be defined within a **statement** block.
  - You can even arbitrarily pull a pair of **curly braces** to define variables.

- The variables within this block **do not exist** until the program enters the block, and they **disappear** after leaving the block.
