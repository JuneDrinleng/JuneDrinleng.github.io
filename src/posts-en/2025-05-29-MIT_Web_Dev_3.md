## 3 JavaScript Operators

For `number` operations like addition, subtraction, multiplication, and division are normal. String concatenation is also standard. Additionally, Boolean generation is largely consistent with Python (using three symbols for inequality and equality).

![image-20250529161259220](https://raw.githubusercontent.com/JuneDrinleng/JuneDrinleng.github.io/main/assets/img/image-20250529161259220.png)

**It is easy to notice here that equality is checked using `===`, and inequality is checked using `!==`.**

We need to discuss the difference between `==` and `===`.

In general programming languages, `==` is used to check if two variables point to the same memory address. In JavaScript, this is achieved using `===`. The general `==` in JavaScript first performs type coercion before comparison, which can lead to results that contradict intuition:

![image-20250529162015499](https://raw.githubusercontent.com/JuneDrinleng/JuneDrinleng.github.io/main/assets/img/image-20250529162015499.png)

For example, `2=="2"` first converts `2` and `"2"` to the same type, e.g., both become `"2"`, then compares them. Since they are equal, it returns `true`.

Due to this mechanism, we always use the triple equals sign `===` for comparisons rather than the double equals `==`.

## 4 JavaScript Basic Syntax
Note that each line of code must end with a semicolon. Unlike Python, JavaScript is indentation-insensitive, and code blocks are identified by curly braces `{}`. Comments are implemented via `//`.

For variable declaration in JavaScript, it is necessary to declare variables using the `let` statement. Variable naming should follow camelCase convention, and underscore-based naming is not recommended.

![image-20250529162851460](https://raw.githubusercontent.com/JuneDrinleng/JuneDrinleng.github.io/main/assets/img/image-20250529162851460.png)

Constants are defined using the `const` statement.

![image-20250529163149528](https://raw.githubusercontent.com/JuneDrinleng/JuneDrinleng.github.io/main/assets/img/image-20250529163149528.png)

Why distinguish between variables and constants? To ensure safe coding and prevent accidental modification of constants that may affect subsequent program logic. Attempting to change a `const` constant will result in an error message.

It is worth noting that there is another variable declaration method `var`. The difference lies in the scope: `var` is limited to the function scope (variables defined within a function), while `let` variables are restricted by curly braces `{}`.

~~~javascript
for (let i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100); // 0, 1, 2
}
// setTimeout is an asynchronous function. If using `var`, the callback will execute after the loop completes, at which point `i` will be 3.
~~~

~~~javascript
function test() {
  if (true) {
    var a = 123;
  }
  console.log(a); // 123, `a` is accessible throughout the entire function
}
~~~

To achieve clearer variable scoping, avoid using `var`.

## 5 null and undefined

`undefined` indicates that no value has been assigned. For example, a variable can be declared without being assigned a value.

`null` represents the absence of a value.

To clearly understand variable values, we can use `console.log()` to achieve an effect similar to `print`.

Implementation of structured strings:

~~~javascript
let a = 1;
let b = 2;
console.log(`a+b=${a+b}`);
~~~

Another debugging method is to use the `alert` function.
