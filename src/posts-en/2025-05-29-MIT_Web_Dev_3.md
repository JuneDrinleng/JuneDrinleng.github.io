Operations such as addition, subtraction, multiplication, and division of `number` are normal. In addition, the addition operation of strings is also normal. In addition, the generation of Boolean is basically the same as python (not equal to and equal to are three symbols).

**It is not difficult to find that `===` is used to judge whether they are equal here, and `!==` is used to judge whether they are equal.**

Here we need to discuss the difference between `==` and `===`

For general programming languages, `==` is generally used to determine whether the two variables point to the same memory address. This is implemented in js through `===`, and the general `==` in js first performs a type consistency operation and then compares it, which will lead to many results that are contrary to intuition:

![image-20250529162015499](https://raw.githubusercontent.com/JuneDrinleng/JuneDrinleng.github.io/main/assets/img/image-20250529162015499.png)

For example, `2=="2"` here will first convert `2` and `"2"` into the same type, for example, both into `"2"` and then compare them. At this time, the two are equal, so the return is `true`

Due to the existence of the above mechanism, we always use the triple symbol when comparing, instead of the double equal sign.

## 4 js basic syntax

Note that each line of statements requires a semicolon at the end. Different from python, js is indent-insensitive. The code block is distinguished from the curly brackets {}, and the comments are implemented through //

For js variable definition, it needs to be declared using the `let` statement. At the same time, the variable naming needs to use camel case naming method. It is not recommended to use the underline naming method.

![image-20250529162851460](https://raw.githubusercontent.com/JuneDrinleng/JuneDrinleng.github.io/main/assets/img/image-20250529162851460.png)

The definition of constants is achieved through the `const` statement

![image-20250529163149528](https://raw.githubusercontent.com/JuneDrinleng/JuneDrinleng.github.io/main/assets/img/image-20250529163149528.png)

Why are there variables and constants? For safe coding, certain constants are prevented from being modified and affecting subsequent programs. If you try to change a const constant, js will return an error message.

It should be noted that in the past, there was another definition method for variable definition, `var`. The difference is that the scope of var is only limited by functions (variables within functions), while the variables of let are limited by curly braces.

~~~javascript
for (let i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100); // 0, 1, 2
}
//setTimeout is an asynchronous function. If it is var, the callback will not be executed until the end of the loop. At this time, the callback is performed after the end of the loop, which means that all i are 3.
~~~

~~~javascript
function test() {
  if (true) {
    var a = 123;
  }
  console.log(a); // 123, a is visible throughout the function
}

~~~

So for clearer variable scope, please don’t use `var`

## 5 null and undefined

`undefined` means that no content is allocated. For example, a variable can be declared without assigning a value.

`null` means there is no value

In order to clearly understand the value of the variable, we can use `console.log()` to achieve an effect similar to `print`

Implementation of structured strings:

~~~javascript
let a=1;
let b=2;
console.log(`a+b=${a+b}`);
~~~

Another way to debug is to use the `alert` function
