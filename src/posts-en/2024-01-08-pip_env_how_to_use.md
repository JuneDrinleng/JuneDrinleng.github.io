# pipenv Usage Guide

pipenv mainly creates a virtual environment under your project, but it faces the following issues:

The first issue is that the newly created environment is located in the C drive, which is inconvenient. The second issue is that the naming is random.

To facilitate usage, we first need to add environment variables.

## 1 Adding Environment Variables

### 1.1 Modifying the System Environment Variable WORKON_HOME

Modify the system environment variable WORKON_HOME. If it does not exist, create it. Set its value to **.**

Explanation: **.** represents the current directory relative to the working directory. After setting this, the virtual environment will be installed under the project's working directory.

![](https://raw.githubusercontent.com/JuneDrinleng/JuneDrinleng.github.io/main/assets/img/fbac368be13c4cd49d8a288b87334c05.png)

### 1.2 Creating and Modifying the System Environment Variable PIPENV_CUSTOM_VENV_NAME

Create and modify the system environment variable PIPENV_CUSTOM_VENV_NAME. Setting this variable ensures that the virtual environment folder is named according to this character variable instead of being randomly generated. For example, if I set it to "venv" as shown in the figure, the final virtual environment folder name will be "venv". Of course, you can rename it to anything else.

![](https://raw.githubusercontent.com/JuneDrinleng/JuneDrinleng.github.io/main/assets/img/5014664d3bfc47b795a1869c4dc87166.png)

## 2 Creating a Virtual Environment

### 2.0 Installing pipenv

Install Python by downloading a specific version from the official website and adding it to the environment variables.

If your computer has not installed pipenv after installing Python, you need to go through this step.

~~~
pip install pipenv
~~~

### 2.1 Navigate to the Virtual Environment Location

First, use CMD to navigate to your working directory. This is because the terminal already has the conda path, and having both together may cause conflicts.

### 2.2 Creating the Virtual Environment

~~~
pipenv install
~~~

### 2.3 Activating the Newly Created Virtual Environment

~~~
pipenv shell
~~~

### 2.4 Installing Dependencies

Note that the requirements here are different from those generated by conda. They cannot be confused.

~~~
pipenv install -r requirements.txt
~~~

Note: How to generate a requirements file

~~~
pip freeze > requirements.txt
~~~

### 2.5 Deleting the Environment

Navigate to the virtual environment location, for example, if the venv folder is the virtual environment, navigate to its parent directory.

~~~
pipenv --rm
~~~
