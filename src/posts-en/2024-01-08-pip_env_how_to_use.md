# pip env usage guide

pip env mainly creates a new virtual environment under your project, but it will face the following problems

One is that the new environment is located on the c drive and is easy to send, and the other is that it is named randomly.

Therefore, for ease of use, first add the environment variables

## 1 Adding environment variables

### 1.1 Modify the system environment variable WORKON_HOME

Modify the system environment variable WORKON_HOME. If it does not exist, create a new one and set its value to **.**

Explanation: . represents the current directory for the working directory. After setting, the virtual environment will be installed in the working directory of the project code.

![](https://raw.githubusercontent.com/JuneDrinleng/JuneDrinleng.github.io/main/assets/img/fbac368be13c4cd49d8a288b87334c05.png)

### 1.2 Create and modify the system environment variable PIPENV_CUSTOM_VENV_NAME

Create and modify the system environment variable PIPENV_CUSTOM_VENV_NAME. After setting its value, the name of the folder where the virtual environment is located is not randomly generated, but based on this character variable. For example, if I set it to venv in the picture below, then the name of my virtual environment folder will be venv. Of course, you can also change this name to something else.

![](https://raw.githubusercontent.com/JuneDrinleng/JuneDrinleng.github.io/main/assets/img/5014664d3bfc47b795a1869c4dc87166.png)

## 2 Create a virtual environment

### 2.0 Install pipenv

To install python, download a certain version of python from the official website and add it to the environment variables.

If you have not installed pipenv after installing python on your computer, you need to go through this step

~~~
pip install pipenv
~~~

### 2.1 cd to the location of the virtual environment

First, the first step is to cd to your working directory. Please use cmd to achieve it, because the terminal has added the conda path, and the two will fight together.

### 2.2 Create a virtual environment

~~~
pipenv install
~~~

### 2.3 Activate the newly created virtual environment

~~~
pipenv shell
~~~

### 2.4 Install dependencies

It should be noted that the requirements here are different from those generated by conda and cannot be confused.

~~~
pipenv install -r requirements.txt
~~~

Note: How to generate requirements file

~~~
pip freeze > requirements.txt
~~~



### 2.5 Delete environment

cd to the location of your virtual environment. For example, the venv folder is a virtual environment, then cd to its upper level.

~~~
pipenv --rm
~~~
