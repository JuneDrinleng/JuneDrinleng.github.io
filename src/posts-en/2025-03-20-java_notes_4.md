## 1 Loops

For the previous ticket purchasing program, the existing code can only buy one ticket and then end the program. However, in reality, the system should repeatedly purchase tickets, which requires the use of loop concepts:

Original code:

~~~java
import java.util.Scanner;

public class compare {
    public static void main(String[] args) {
        System.out.print("Please insert a certain amount of money:");
        Scanner in = new Scanner(System.in);
        int amount=in.nextInt();
        if (amount>=10){
            System.out.println("Change: "+(amount-10));
            System.out.println("************************");
            System.out.println("Java Inter-city Railway Ticket");
            System.out.println("Fare: 10 yuan");
            System.out.println("************************");
        }
        else {
            System.out.println("Insufficient amount");
        }
    }
}

~~~

As observed, the system only outputs the ticket once and then ends, which is not suitable for practical use:

![image-20250320221215945](https://raw.githubusercontent.com/JuneDrinleng/JuneDrinleng.github.io/main/assets/img/image-20250320221215945.png)

To address this, we need to make the above process repeat, which can be achieved using the while statement:

~~~
import java.util.Scanner;

public class compare {
    public static void main(String[] args) {
        System.out.print("Please insert a certain amount of money:");
        while(True){
            Scanner in = new Scanner(System.in);
            int amount=in.nextInt();
            if (amount>=10){
                System.out.println("Change: "+(amount-10));
                System.out.println("************************");
                System.out.println("Java Inter-city Railway Ticket");
                System.out.println("Fare: 10 yuan");
                System.out.println("************************");
            }
            else {
                System.out.println("Insufficient amount");
            }
       }
    }
}

~~~

This is the runtime result:

![image-20250320221502422](https://raw.githubusercontent.com/JuneDrinleng/JuneDrinleng.github.io/main/assets/img/image-20250320221502422.png)

It is evident that our program faces several issues. First, when the single amount is insufficient but multiple coin insertions eventually meet the requirement, the ticket cannot be purchased. Second, the program will never terminate.

To address the coin accumulation, it's simple to add a cumulative variable:

~~~java
import java.util.Scanner;

public class compare {
    public static void main(String[] args) {
        Scanner in = new Scanner(System.in);
        int balance=0;
        while (true) {
            System.out.print("Please insert a certain amount of money:");
            int amount = in.nextInt();
            balance=balance+amount;
            if (balance >= 10) {
                System.out.println("Change: " + (balance - 10));
                System.out.println("************************");
                System.out.println("Java Inter-city Railway Ticket");
                System.out.println("Fare: 10 yuan");
                System.out.println("************************");
                balance=0;
            } else {
                System.out.println("Insufficient amount");
            }
        }
    }
}

~~~

## 2 Counting Numbers
~~~
import java.util.Scanner;

public class CountNumber {
    public static void main(String[] args) {
        Scanner in = new Scanner(System.in);
        System.out.println("Welcome to Digit Counting");
        System.out.print("Please enter a number to check: ");
        int amount= in.nextInt();
        int count=0;
        while (amount>0)
        {
            amount=amount/10;
            count=count+1;
        }
        System.out.println(count+" digits");
    }
}

~~~

It can be implemented by continuously dividing by 10

The parentheses after "while" in the code represent the condition for the while loop to run. If the condition is met, the loop will execute continuously. Note that the loop body will only return to check the condition after completing one iteration

## 4 do-while Statement

For the above scenario, we can first execute the loop content and then check the condition, which can be achieved using a do-while statement

~~~
do{
	operation;
}while(condition);
~~~

Special attention should be paid to the semicolon after "while" in the do-while loop

## 5 Counting Loop

Counting loops are used to control how many times the loop executes

![image-20250321104955129](https://raw.githubusercontent.com/JuneDrinleng/JuneDrinleng.github.io/main/assets/img/image-20250321104955129.png)

This actually runs 101 iterations, with the final count outputting -1, which is related to the condition "count >= 0". If we change it to "count > 0", the loop will run 100 times, and the last output will be 0

Simultaneously, whether to print first or decrement count by 1 affects the output range. For example, it will start printing from 100 if we do the latter

When unsure about the number of iterations, it's advisable to test with a smaller loop count first

## 6 Task: Calculate Average

![image-20250321105721518](https://raw.githubusercontent.com/JuneDrinleng/JuneDrinleng.github.io/main/assets/img/image-20250321105721518.png)

~~~
import java.util.Scanner;

public class AverageCalculate {
    public static void main(String[] args) {
        int count=0,sum=0,number;
        Scanner in = new Scanner(System.in);
        do {
            System.out.print("Enter a number: ");
            number=in.nextInt();
            count=count+1;
            sum=sum+number;
        } while (number!=-1);
        if (count-1>0) {
            double average=(sum+1)*1.0/(count-1);
            System.out.println("Average is: "+average);
        }

    }
}

~~~

Some details: If the input starts with -1, count will be 0 initially

## 7 Task: Guessing Game

Task: Let the computer think of a number, then let the user guess. After each input, inform the user if their guess is too high or too low until they guess correctly, then output the number of attempts

The problem can be broken down into the following parts:

- Generate a random number for the computer
- Initialize a counter variable to 0
- Let the user input a number
- Increment the counter
- Compare the input with the random number. If the random number is larger, output "Too big", otherwise "Too small"
- If the input doesn't match the random number, return to let the user input again. If they match, output "Correct" along with the attempt count and end the program

~~~
// Random number generation
Math.random() // Generates a number in [0,1)
~~~

The final code is shown in the figure below:

~~~
import java.util.Scanner;

public class GuestingNumber {
    public static void main(String[] args) {
        int number=(int)(Math.random()*100+1);
        Scanner in= new Scanner(System.in);
        int input_num,count=0;
        do {
            System.out.print("Enter a number between 1-100: ");
            input_num=in.nextInt();
            count=count+1;
            if (input_num>number){
                System.out.println("Too big");
            }
            else if (input_num<number){
                System.out.println("Too small");
            }
        }while(input_num != number);
        System.out.println("Correct, guessed "+count+" times");
    }
}

~~~

## 8 Task: Integer Decomposition
Using %10 can extract the units digit, while /10 can remove the units digit

~~~
import java.util.Scanner;

public class SplitInt {
    public static void main(String[] args) {
        Scanner in= new Scanner(System.in);
        System.out.print("Enter a number: ");
        int num = in.nextInt();
        int mod;
        do{
            mod=num%10;
            num=num/10;
            System.out.print(mod);
        }while (num>0);
    }
}

~~~
