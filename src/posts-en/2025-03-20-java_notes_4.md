## 1 loop

For the previous ticket buying program, the existing program can only buy one ticket and the entire program is over. However, in reality, tickets should be purchased again and again. This requires the use of loop-related knowledge:

The original code is as follows:

~~~java
import java.util.Scanner;

public class compare {
    public static void main(String[] args) {
        System.out.print("Please invest a certain amount of currency:");
        Scanner in = new Scanner(System.in);
        int amount=in.nextInt();
        if (amount>=10){
            System.out.println("Change: "+(amount-10));
            System.out.println("************************");
            System.out.println("java intercity railway ticket");
            System.out.println("Fare: 10 yuan");
            System.out.println("************************");
        }
        else {
            System.out.println("The amount you invested is insufficient");
        }
    }
}

~~~

It can be seen that the entire system ends after outputting the ticket only once, which is not suitable for actual use:

![image-20250320221215945](https://raw.githubusercontent.com/JuneDrinleng/JuneDrinleng.github.io/main/assets/img/image-20250320221215945.png)

In order to deal with this content, we need to let the above process happen repeatedly. In this case, we can use the while statement:

~~~
import java.util.Scanner;

public class compare {
    public static void main(String[] args) {
        System.out.print("Please invest a certain amount of currency:");
        while(True){
            Scanner in = new Scanner(System.in);
            int amount=in.nextInt();
            if (amount>=10){
                System.out.println("Change: "+(amount-10));
                System.out.println("************************");
                System.out.println("java intercity railway ticket");
                System.out.println("Fare: 10 yuan");
                System.out.println("************************");
            }
            else {
                System.out.println("The amount you invested is insufficient");
            }
       }
    }
}

~~~

This is the result of running:

![image-20250320221502422](https://raw.githubusercontent.com/JuneDrinleng/JuneDrinleng.github.io/main/assets/img/image-20250320221502422.png)

It is not difficult to find that our program faces the following problems. First, the amount is insufficient once but the amount is reached after multiple coin inputs, but the corresponding ticket cannot be purchased. Second, our program will never stop.

In order to deal with the accumulation of change, it is actually very simple, just add one more accumulated variable:

~~~java
import java.util.Scanner;

public class compare {
    public static void main(String[] args) {
        Scanner in = new Scanner(System.in);
        int balance=0;
        while (true) {
            System.out.print("Please invest a certain amount of currency:");
            int amount = in.nextInt();
            balance=balance+amount;
            if (balance >= 10) {
                System.out.println("Change: " + (balance - 10));
                System.out.println("************************");
                System.out.println("java intercity railway ticket");
                System.out.println("Fare: 10 yuan");
                System.out.println("************************");
                balance=0;
            } else {
                System.out.println("The amount you invested is insufficient");
            }
        }
    }
}

~~~

## 2 Count numbers

~~~
import java.util.Scanner;

public class CountNumber {
    public static void main(String[] args) {
        Scanner in = new Scanner(System.in);
        System.out.println("Welcome to judging the number of digits");
        System.out.print("Please enter the number to be judged: ");
        int amount= in.nextInt();
        int count=0;
        while (amount>0)
        {
            amount=amount/10;
            count=count+1;
        }
        System.out.println(count+"number of digits");
    }
}

~~~

Just keep dividing by 10 to achieve it

The brackets after `while` here are the conditions for the while loop to be established. If the conditions are established, the loop will always be running. It should be noted that only after the loop body ends once, it will return to the place where the condition is judged to judge whether the next loop will proceed.

## 4 do-while statement

For the above situation, we can first do the content of the loop and then make a judgment. In this case, we can use the do-while statement

~~~
do{
	operation;
}while(condition);
~~~

Pay special attention to the semicolon after while in the do-while loop

## 5 Counting loop

Counting loops are used to control how many times the loop is executed

At the same time, whether `print` or `count-1` comes first will also affect the range of our output. For example, `print` will start from 100 at this time.

## 6 task: Calculate the average

![image-20250321105721518](https://raw.githubusercontent.com/JuneDrinleng/JuneDrinleng.github.io/main/assets/img/image-20250321105721518.png)

~~~
import java.util.Scanner;

public class AverageCalculate {
    public static void main(String[] args) {
        int count=0,sum=0,number;
        Scanner in = new Scanner(System.in);
        do {
            System.out.print("Please enter a number:");
            number=in.nextInt();
            count=count+1;
            sum=sum+number;
        } while (number!=-1);
        if (count-1>0) {
            double average=(sum+1)*1.0/(count-1);
            System.out.println("The average is: "+average);
        }

    }
}

~~~

Some small details: when it is -1 at the beginning, then the count is 0 at this time

## 7 task: Guessing game

Task: Let the computer think of a number, and then let the user guess it. Every time the user enters a number, tell him whether it is larger or smaller. Until the user guesses correctly, output how many times the user has guessed.

Then the problem can be broken down into the following parts:

- The computer randomly generates a number
- Count variable initialized to 0
- Let the user enter a number
- Counter iteration +1
- Determine the size of a and the random number. If the random number is large, the output will be large, otherwise the output will be small.
- If a and the random number are not equal, the program will return to asking the user to enter the number, otherwise it will output the guess and the number of times, and finally end

~~~
//Generation of random numbers
Math.random() //What is generated here is [0,1);
~~~

The final code is shown below:

~~~
import java.util.Scanner;

public class GuestingNumber {
    public static void main(String[] args) {
        int number=(int)(Math.random()*100+1);
        Scanner in= new Scanner(System.in);
        int input_num,count=0;
        do {
            System.out.print("Please enter a number between 1-100:");
            input_num=in.nextInt();
            count=count+1;
            if (input_num>number){
                System.out.println("Too big");
            }
            else if (input_num<number){
                System.out.println("Too small");
            }
        }while(input_num != number);
        System.out.println("Guessed correctly, guessed "+count+" times");
    }
}

~~~
